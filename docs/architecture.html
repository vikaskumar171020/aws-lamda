Project Architecture

Detailed architecture breakdown of the serverless Express.js app running inside AWS Lambda.

Architecture Topology

Below is the data execution lifecycle. A client request triggers an API Gateway resource proxy which invokes the AWS Lambda runtime node process, wrapping raw payload structures for Express standard routes.

Key Architectural Components

1. AWS SAM Framework (template.yaml)

AWS Serverless Application Model (SAM) provides the resource definitions to build, package, and orchestrate serverless applications.

  • ExpressApiFunction: Defines the single Lambda function container that acts as the entry handler.
  • Wildcard Event Integration: A proxy resource (/{proxy+}) and root resource (/) catch all incoming endpoints, methods, and parameters from API Gateway and forward them as event objects to the node function.
  • TypeScript Compilation: The TypeScript build script uses esbuild to pack, bundle, and tree-shake libraries, producing a lightweight distribution file to optimize cold start profiles.

2. Express Routing App (src/app.ts)

Standard Express.js acts as the backend routing core, facilitating familiar controller structures, middleware interceptors, and local error handling mechanisms. This approach matches standard Node.js API structures, enabling easy migration paths between traditional server-hosted architectures and serverless runtimes.

3. Serverless HTTP Adaptor (src/handlers/api.ts)

The serverless-http interface wraps the Express server structure, mapping the AWS API Gateway event context arguments (headers, queries, params, body) into standard Express request nodes, and translates the returning Express output structures back into API Gateway JSON payloads.
