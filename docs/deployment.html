Deployment Guide

Instructions to deploy the AWS CloudFormation stack locally and configure secure CI/CD workflows.

First-Time Guided Deployment

Configure project dependencies, regions, and parameters interactively using AWS SAM CLI:

npm run build sam deploy --guided

Guidance Prompts

  1. Stack Name: Select the CloudFormation project stack tag (default: aws-lambda-express-api).
  2. AWS Region: The region code where resources should be instantiated (e.g. us-east-1).
  3. Confirm changes before deploy: Set to Y to review resource diffs before they are provisioned.
  4. Allow SAM CLI IAM role creation: Set to Y so SAM can configure execution permissions.
  5. Disable rollback: Set to N to automatically roll back stack changes if a validation error occurs.
  6. ExpressApiFunction authorization warning: Select Y (standard public API routing, endpoints can be secured by custom middleware inside Express).
  7. Save arguments to configuration file: Set to Y to save settings to samconfig.toml.

Subsequent Stack Deployments

Once your samconfig.toml configuration parameters are defined, subsequent deployments only require executing:

npm run build sam deploy

Continuous Integration & Delivery (CI/CD)

Deployments are automated through two distinct GitHub Actions workflows:

  • Validation Workflow (validate.yml): Runs automatically on pull requests to main to run tests, type-checks, and builds.
  • Manual Deployment Workflow (deploy.yml): Triggered manually on release. Increments versioning, bundles files into zip archives, creates Git Releases, and triggers SAM Deploy.

AWS CloudFormation provides transactional rollback. If a deployment script encounters an error, CloudFormation aborts execution and rolls all infrastructure resources back to the last stable state.

GitHub Actions AWS OIDC Integration

Set up passwordless OpenID Connect authentication to securely build resources from GitHub without committing static credentials:

  1. AWS IAM Provider Configuration: Add an OpenID Identity Provider mapping URL https://token.actions.githubusercontent.com with audience sts.amazonaws.com.
  2. Create Role Assignment: Set up a new role configuration mapping your repository name to web identity tokens, attaching policies to interact with S3, Lambda, and CloudFormation.
  3. Repository Secrets: Define the role ARN inside GitHub Secrets under AWS_ROLE_TO_ASSUME.
