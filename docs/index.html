Observability Insights Hub

Get actionable details about application behavior. Select a metric category to view pre-built CloudWatch Logs Insights queries.

This CloudWatch Logs Insights query extracts execution stats to calculate the total percentage of requests experiencing cold starts (initialization delays).

filter @type = "REPORT" | stats count() as TotalInvocations, count(@initDuration) as ColdStarts, (count(@initDuration)/count())*100 as ColdStartPct | sort ColdStartPct desc

Analyze performance profiles across p50, p90, and p99 percentiles to track response distribution and diagnose slow executions.

filter @type = "REPORT" | stats pct(@duration, 50) as p50, pct(@duration, 90) as p90, pct(@duration, 99) as p99 by bin(1h) | sort @timestamp desc

Identify error conditions, application crashes, uncaught exceptions, and bad request statuses (4xx/5xx) occurring inside the container handler.

filter @message like /error/ or @message like /Error/ or @message like /404/ or @message like /500/ | fields @timestamp, @message, @logStream, @requestId | sort @timestamp desc | limit 50

Audit provisioned memory settings. This helps prevent over-provisioning (wasting budget) and under-provisioning (triggering performance slowdowns).

filter @type = "REPORT" | stats max(@maxMemoryUsed) as MaxMemoryUsed, avg(@maxMemoryUsed) as AvgMemoryUsed, max(@memorySize) as ProvisionedMemory | sort @timestamp desc

Optimal Performance Guidelines

Enable AWS Lambda Insights Layer

Deploy the CloudWatch Lambda Insights extension layer to capture enhanced operating system metrics, including precise CPU memory workloads and local network I/O.

Optimize Dependency Bundle (Webpack/Esbuild)

This project uses esbuild to bundle the API code. Keep bundling minification active to ensure file sizes are small, keeping cold start file-loading times under 150ms.

Provisioned Concurrency vs On-Demand

If p99 user latency is a critical business metric, provision pre-warmed instances to eliminate Cold Starts completely during high-traffic windows.
