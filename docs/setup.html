Local Development Setup

Follow these instructions to configure and run your serverless Express API on your local machine.

Prerequisites

Ensure you have the following resources installed on your development workstation:

  • Node.js: Version 20.x or higher is recommended for compilation and runtime.
  • npm: Node package manager (bundled natively with Node.js).
  • AWS CLI: Configured with appropriate AWS developer credentials.
  • AWS SAM CLI: Command Line Interface for local emulation, building, and deployments.
  • Docker: Required by AWS SAM to host local containerized Lambda execution environments.

1. Install Project Dependencies

Run the package manager from the root directory to configure the environment, typescript interfaces, and dev tools:

npm install

2. Build and Compile TypeScript

Use AWS SAM to build the application. This compiles code via esbuild based on the rules specified in your template.yaml file:

npm run build

3. Run Local Emulation Server

Launch the local API Gateway. AWS SAM emulates the environment inside a Docker container, routing events directly to your Express app:

npm run local

Your API will now be listening on http://127.0.0.1:3000. You can query endpoints via your browser, Postman, or terminal utilities.

4. Run Unit Tests

Execute tests using Jest and Supertest to verify controller routes and validation criteria:

npm test

To run coverage reports ensuring at least 80% code coverage is maintained:

npm run test:coverage

5. Verification & Type Checking

Verify that there are no TypeScript syntax or compilation discrepancies in the codebase:

npm run type-check
